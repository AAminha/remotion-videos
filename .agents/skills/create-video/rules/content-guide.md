# 콘텐츠 가이드

주제/맥락을 영상 씬 구성으로 변환하는 가이드.

---

## 기본 원칙

1. **hero로 시작**: 모든 영상은 hero 씬으로 시작한다
2. **5~9개 씬**: 너무 짧으면 정보 부족, 너무 길면 집중력 저하
3. **내용에 맞는 템플릿만**: 다양한 템플릿을 골고루 쓰지 말고, 주제에 맞는 것만 선택
4. **duration은 기본값 사용**: TTS가 오디오 길이에 맞춰 자동 조정함
5. **한국어 콘텐츠**: 본문은 한국어, 영어 기술 용어는 그대로 사용 (TTS 발음 사전이 처리)

---

## 주제 유형별 추천 구성

### 기술 소개 (예: "Why TypeScript?")

```
hero     → 기술명 + 한 줄 요약
list     → 핵심 장점/특징
code     → 코드 예시
stat     → 채택률/인기도 수치
flow     → 개발 워크플로우
chat     → 팀 내 도입 대화 (선택)
grid     → 관련 생태계/도구 (선택)
```

### 아키텍처/시스템 설명 (예: "BigStream × BytePlus 파이프라인")

```
hero     → 시스템명 + 한 줄 요약
flow     → 전체 흐름 미리보기
sequence → Phase별 상세 호출 흐름 (여러 개)
list     → 주의사항/실패 케이스
flow     → 에러 핸들링 흐름 (선택)
```

### 작업 브리핑 (예: "새벽 마이그레이션 작업")

```
hero     → 작업명 + 일시
list     → 배경/이유
list     → 예상 영향
flow     → 작업 타임라인
list     → 모니터링 방법
list     → 문제 시 대응
hero     → 완료 기준 (마무리)
```

### 온보딩/가이드 (예: "신규 개발자 가이드")

```
hero     → 환영 메시지
list     → 알아야 할 것들
flow     → 셋업 절차
code     → 핵심 코드 패턴
chat     → FAQ (Q&A 형식)
grid     → 참고 리소스
```

### 회고/리뷰 (예: "Q1 스프린트 회고")

```
hero     → 기간 + 주제
stat     → 핵심 지표
list     → 잘한 점
list     → 개선할 점
flow     → 다음 스프린트 계획
```

## 씬 타입 선택 기준

| 콘텐츠 성격               | 추천 씬 타입 |
| ------------------------- | ------------ |
| 제목/시작/끝              | hero         |
| 항목 나열 (3개 이상)      | list         |
| 카드형 정보 (아이콘+설명) | grid         |
| 코드 보여주기             | code         |
| 순서가 있는 절차          | flow         |
| 대화/토론                 | chat         |
| 숫자/통계 강조            | stat         |
| 시스템 간 호출 흐름       | sequence     |

## 콘텐츠 작성 팁

### 제목

- 짧고 명확하게 (15자 이내 권장)
- hero의 subtitle로 부연 설명

### list 항목

- 한 항목당 한 문장
- 핵심만 담기 (TTS가 전부 읽으므로 너무 길면 지루함)
- 최대 8개, 보통 3~5개가 적당

### flow 단계

- label은 짧게 (2~4단어)
- description으로 보충
- 2~7단계, 보통 3~5단계가 적당

### chat 메시지

- 자연스러운 대화체
- isUser: true/false로 화자 구분 (TTS 음성이 달라짐)
- 4~6개 메시지가 적당

### sequence 메시지

- actors는 시스템/서비스 이름
- label은 API 호출명이나 행위
- 한 씬에 최대 12개 메시지

### code 블록

- 핵심 부분만 (전체 코드 X)
- 주석으로 포인트 강조
- TTS는 title만 읽으므로 title에 코드의 핵심을 담기

## 입력 유형별 대응

### 주제만 주어진 경우

"Kafka 소개 영상 만들어줘"
→ 해당 기술에 대한 지식을 활용하여 구성 자동 생성

### 맥락/내용이 정리된 경우

텍스트나 bullet point로 내용 공유
→ 내용을 씬 타입에 매핑하여 구성

### 파일이 공유된 경우

마크다운, 노션 페이지, 슬랙 메시지 등
→ 파일 내용을 분석하여 핵심 추출 후 구성

### 기존 영상 수정 요청

"TypeScript 영상에 테스트 씬 추가해줘"
→ 기존 video-config.json 수정 + TTS 재생성
